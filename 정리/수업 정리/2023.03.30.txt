메타 데이터
=> Class와 같이 데이터들을 정의해놓은 형태

시스템이 어떤 흐름을 가지는지 알 수 있는 것을 ERD라고 한다

* ERD
Entity (개체) 사각형
실제로 존재하는 것
ex) 열차, 정거장

Relation (관계) 마름모
개체들 간의 관계
ex) 출발한다, 도착한다

Attribute (속성) 타원
개체의 필드, 속성
기본 키로 쓰이는 속성은 밑줄을 긋는다
ex) 열차 번호

실선 : 참조하는 필드가 PK로 사용될 경우 실선 표시 
점선 : 참조하는 필드가 PK로 사용하지 않을 경우 점선 표시

조인을 사용하면 검색 속도에 많은 영향을 미친다
꼭 조인을 쓰지 않아도 되는 상황이면 서브쿼리를 쓰는 것이 검색 속도가 더 빠르다
조인을 쓸 때 : 두 개의 테이블에서 데이터를 가져와서 하나의 결과물을 보여줄 때 사용

서브쿼리를 쓸 때 : (조건에서 사용) 다른 테이블에서 조건을 검색하여 검색된 값을 가져와서 테이블의 조건으로 사용할 때 사용
(FROM 절에서 사용) 연산을 한 후 검색을 하고 싶으면 FROM 절에 서브쿼리를 쓴다 FROM 절에 서브쿼리를 쓸 때는 무조건 별칭을 적어줘야 한다

정규화
관계형 데이터베이스에서 신뢰성을 높인다
검색 속도와 데이터 수정이 어렵다

일시적인 데이터는 반정규화를 사용하여 정규 허용
