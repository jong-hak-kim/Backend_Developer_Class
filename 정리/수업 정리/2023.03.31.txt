*인덱스
1. 인덱스를 이용하면 테이블 전체를 읽을 필요 없이 해당 인덱스를 찾아가기만 하면 되므로 검색과 질의 처리에 빠르다
2. 데이터 삽입, 삭제, 수정 작업이 원본 테이블에서 하면, 인덱스도 수정되어야 하므로 인덱스가 있는 필드는 수정을 하지 않는 것이 좋다

CREATE INDEX saedaesu_index
ON Namgu (세대수); #오로지 세대수로만 검색할 때 이렇게 사용

CREATE INDEX saedaesu_index_2
ON Namgu (세대수, 인구수, 통);
# 여러 개의 필드를 인덱스로 정해뒀을 때 자주 사용하는 필드들을 앞으로 정해두면 좋다

*데이터베이스 생성 순서

1. ERD 만들기
개체(Entity) 찾기
ex) 회원(User), 게시물(Board)

관계(Relation) 찾기
회원이 게시물을 작성한다
게시물이 회원에 종속
1 : N 관계

회원이 게시물을 좋아한다
회원이 여러 게시물을 좋아요할 수 있다
게시물도 여러 회원에게 좋아요를 받을 수 있다
N : M 관계

회원이 댓글을 단다
회원이 여러 게시물을 댓글달 수 있다
게시물도 여러 회원에게 댓글을 받을 수 있다
N : M 관계

- 로그인 화면
유저 (이메일 주소, 비밀번호)

- 회원 가입
유저 (이메일 주소, 비밀번호, 비밀번호 확인, 닉네임, 휴대전화 번호, 주소, 상세 주소, 개인정보처리 동의)

- 메인 화면
1. 주간 Top3 게시물
게시물 (게시물 번호, 게시물 사진, 작성자 프로필 사진, 작성자 닉네임, 작성 날짜, 제목, 내용, 조회수, 좋아요 수, 댓글 수)

2. 최신 게시물
게시물 (게시물 번호, 게시물 사진, 작성자 프로필 사진, 작성자 닉네임, 작성 날짜, 제목, 내용, 조회수, 좋아요 수, 댓글 수)

3. 인기 검색어
검색 로그 (검색어)

- 검색 화면
1. 검색결과
게시물 (게시물 사진, 작성자 프로필 사진, 작성자 닉네임, 작성 날짜, 제목, 내용, 조회수, 좋아요 수, 댓글 수)

2. 관련 검색어
검색 로그 (관련 검색어)

- 상세 페이지
게시물 (게시물 제목, 작성자 이메일, 작성자 프로필 사진, 작성자 닉네임, 작성일, 내용, 게시물 사진, 좋아요 수, 댓글 수)
좋아요 리스트 (유저 프로필 사진, 유저 닉네임)
댓글 리스트 (유저 프로필 사진, 유저 닉네임, 작성 시간, 댓글 내용)

- 마이 페이지
1. 유저 정보
유저 (프로필 사진, 닉네임, 이메일)

2. 내 게시물
게시물 (게시물 번호, 게시물 제목, 작성자 프로필 사진, 작성자 닉네임, 작성일, 내용, 게시물 사진, 좋아요 수, 댓글 수)

- 게시물 작성 화면
게시물 (제목, 내용, 게시물 사진)

- 게시물 수정 화면
게시물 (제목, 내용, 게시물 사진)
